### Reference Documents (`references/`)

Architecture decisions, protocol docs, API notes, and technical deep-dives.

Drop Markdown files here for any reference material that doesn't fit the categories below.

### Notebook LLM Exports (`notebook-llm/`)

Research notes, analysis, and summaries generated by [Google Notebook LLM](https://notebooklm.google/).

- **Markdown files** (`.md`) are tracked in git
- **PDF exports** (`.pdf`) are **gitignored** — upload these to [GitHub Releases](https://github.com/Z0mb13V1/mindcraft/releases) and link from the Markdown summary

**Why?** PDFs are large binary files that bloat git history. GitHub Releases provides free, versioned hosting for binaries.

### Mod Pack Research (`mod-packs/`)

Fabric/Forge mod pack compatibility research, performance benchmarks, and mod interaction matrices.

## Workflow

### Adding a new Notebook LLM export

1. Export as Markdown from Notebook LLM → save to `docs/notebook-llm/`
2. If you also have a PDF: upload to GitHub Releases, add a link in the `.md` file
3. Add an entry to this index

### Adding a reference document

1. Write or save to `docs/references/`
2. Add an entry to this index

### Adding mod pack research

1. Save to `docs/mod-packs/`
2. Add an entry to this index
